# Дискуссия об уменьшении когнитивных искажений (Debiasing Discussion)

> [!NOTE]
> Этот документ представляет собой запись дискуссии о влиянии LLM на когнитивные процессы человека и способах борьбы с искажениями через архитектуру Membria.

## 1. Про Систему 2 Канемана и LLM

### Как LLM вредят Системе 2:
Система 2 — это медленное, энергозатратное, осознанное мышление. Она требует усилий и включается, когда Система 1 (быстрая, автоматическая) не справляется или когда человек сознательно решает «подумать».

**Проблема: LLM систематически отключают Систему 2:**
- **Мгновенное удовлетворение (Instant Gratification):** мозг получает дофамин от «готового ответа» и отключает энергозатратную проверку. Зачем напрягать Систему 2, если чатик уже всё решил?
- **Когнитивная разгрузка (Cognitive offloading):** человек буквально аутсорсит рассуждение модели. Система 2 атрофируется как мышца без нагрузки.
- **Иллюзия глубины:** LLM генерирует текст, который выглядит как результат работы Системы 2 (структура, аргументы, взвешенность). Мозг принимает это за «уже проверено» и не включает собственную проверку.
- **Скорость:** Система 2 медленная по дизайну. LLM отвечает за секунды. Интерфейс не даёт времени на включение медленного мышления.

Исследование **Zhou et al. (ICSE 2026)** подтверждает: разработчики в режиме «Cursor предлагает — я принимаю» переключаются в режим микрорешений Системы 1. Отсюда **29% откатов**.

### Как Membria усиливает Систему 2:
- **Принудительная пауза:** чтобы зафиксировать решение, нужно явно сформулировать логику рассуждений (reasoning), альтернативы, уверенность (confidence). Это заставляет включить Систему 2.
- **Структура вместо потока:** вместо «чатик сказал — я согласился» появляется формат: «какие альтернативы? на чём основано? насколько уверен?». Это чек-лист для Системы 2.
- **Отложенная обратная связь:** Система 2 учится через калибровку. Membria связывает «что думал тогда» с «что получилось» через месяцы. Это именно тот цикл обратной связи (feedback loop), который нужен для улучшения медленного мышления.
- **Разрыв резонанса:** когда Система 1 человека резонирует с «Системой 1» LLM (обе быстрые, обе подвержены искажениям), Membria создаёт внешнюю точку, которая не участвует в этом резонансе.

## 2. Активация Системы 2

Чтобы переключиться с Системы 1 на Систему 2 (по Stanovich et al., 2016), нужно два условия:
1. **Обнаружение необходимости переопределения (Override detection):** распознать, что Система 1 даёт неадекватный ответ (активировать «сигнал тревоги»).
2. **Когнитивное разделение (Cognitive decoupling):** запустить симуляцию альтернатив, отделить текущую ситуацию от автоматического ответа.

**LLM ломают оба условия:**
- **Override detection** не срабатывает, потому что ответ LLM выглядит как результат анализа. Мозг не получает «сигнал тревоги».
- **Cognitive decoupling** не происходит, потому что скорость ответа не даёт времени на симуляцию, и возникает фиксация на первом предложенном направлении.

**Membria восстанавливает эти условия:**
- **Override detection:** явное требование указать альтернативы и уверенность — это формальный триггер.
- **Cognitive decoupling:** структура объекта решения заставляет отделить решение от автоматической реакции.

## 3. Статистика активации Системы 2 при работе с LLM

По данным исследования **Zhou et al. (ICSE 2026)** (808 взаимодействий):
- **56.4%** действий с LLM содержали когнитивные искажения.
- **29.5%** действий пришлось откатывать.

**Конкретные искажения:**
- **Instant Gratification:** 19.3% действий (копирование ответа без проверки).
- **Suggester Preference:** 12.6% действий (принятие без вопросов, даже зная ограничения).
- **Fixation:** вызывает 43.4% откатов (упорное следование первому направлению LLM).

## 4. Специфика человеческого мышления (Baseline)

Даже до эпохи LLM большинство людей жили преимущественно на Системе 1.
По результатам **Cognitive Reflection Test (CRT)**:
- У студентов MIT только **~50%** дали правильные ответы на все вопросы (способность подавить Систему 1).
- В общей популяции — только **20-30%** способны стабильно включать Систему 2.

**Вывод:** Чатик просто убрал последние поводы напрягаться. Раньше Система 2 включалась хотя бы от «трения» (нужно было искать информацию самому). LLM убрал это трение.

---

## 5. Висцеральные влияния

**Висцеральные влияния (Visceral influences)** — это влияние телесных состояний на решения: голод, усталость, эмоции. В Membria это учитывается как один из факторов, отключающих Систему 2. Для простоты мы называем это «влиянием эмоций и физиологии».

---

## 6. Архитектурные апдейты для Debias

Для борьбы с «резонансом искажений» (человек + LLM) архитектура Membria дополняется следующими слоями:

### Уровень 1: Ограниченное извлечение (Constrained Extraction)
Извлекать только факты, а не намерения. LLM не должна «интерпретировать» решение на этапе захвата, чтобы не добавлять собственный bias.

### Уровень 2: Состязательный анализ (Adversarial Analysis)
Для решений с высокой ценностью информации (high-VoI) использовать два вызова LLM:
- **Advocate:** аргументы «ЗА».
- **Devil's Advocate:** аргументы «ПРОТИВ» и поиск слабых мест (assumptions).

### Уровень 3: Человеческое решение с полезным «трением» (Friction by Design)
Для критических решений интерфейс блокирует мгновенное подтверждение, требуя заполнения обязательных полей (альтернативы, предположения, уровень уверенности).

### Уровень 4: Калибровочная петля
Отдельное отслеживание точности для решений, принятых с помощью ИИ (LLM-assisted) vs. только человеком. Это позволяет детектировать «резонансные» ошибки.

**Ключевой инсайт:**
Membria не «исправляет» ИИ изнутри. Она делает искажения ИИ видимыми и измеримыми, создавая внешнюю инфраструктуру измерения поверх обоих участников процесса.
